🔥 大模型前沿

EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments
👥 Jundong Xu, Qingchuan Li, Jiaying Wu
大型语言模型（LLM）代理在各种基准测试中表现出色，但大多数评估假设环境是静态的。相比之下，现实世界中的部署本质上是动态的，要求代理不断调整其知识、技能和行为以适应变化的环境和更新的任务条件。为了解决这一差距，我们引入了EvoArena，这是一个将环境变化建模为终端、软件和社会领域中逐步更新序列的基准套件。我们进一步提出了EvoMem，这是一种基于补丁的记忆范式，通过结构化的更新历史记录记忆的演变，使代理能够通过记忆的变化推理环境的演化。实验表明，当前的代理在EvoArena上表现不佳，在演化的终端、软件和社会偏好领域平均准确率为39.6%。EvoMem持续提升了性能，在EvoArena上平均提高了1.5%，同时也在标准基准GAIA和LoCoMo上分别提高了6.1%和4.8%。除了单个任务外，EvoMem还使EvoArena上的链级准确率提高了3.7%，其中成功需要完成一系列连续相关的进化子任务。机制分析表明，EvoMem改善了记忆中的证据捕获，表明更好地保留了完整的演化环境状态。我们的结果强调了在评估和记忆中建模演化对于可靠代理部署的重要性。 核心贡献：本文提出了EvoMem，一种基于补丁的记忆范式，用于追踪和利用记忆演化，从而提升大语言模型代理在动态环境中的鲁棒性与任务表现。

Influcoder: Distilling Decoders' Gradient Influence Rankings into an Encoder for Data Attribution
👥 Dimitri Kachler, Damien Sileo, Pascal Denis
随着大型语言模型（LLMs）能力的增长，人们越来越重视通过过滤训练数据中的样本来构建高质量的数据集。一般来说，数据归因（Data Attribution, DA）方法旨在估计训练数据集中单个样本如何预设模型以生成特定输出。例如，人们可能想知道哪些数据样本在训练后可能导致了模型的有毒行为。许多方法通过影响函数范式来量化这种条件关系。尽管这类方法在功能上是有效的，但它们缺乏足够的处理速度和存储紧凑性，难以在大规模数据集上实际应用。我们提出了一种名为 Influcoder 的方法，它是一种快速且经济高效的大规模基于影响的数据归因方法。 核心贡献：Influcoder 提出了一种高效的、可扩展的基于影响函数的数据归因方法，解决了传统方法在处理大规模数据时的速度和存储瓶颈问题。

HyperTool: Beyond Step-Wise Tool Calls for Tool-Augmented Agents
👥 Yaxin Du, Yifan Zhou, Yujie Ge
工具增强的LLM代理通常依赖于逐步的原子工具调用，其中每次调用、观察和值传递都会在主推理轨迹中显式呈现。这导致了一种“执行粒度不匹配”问题：局部确定性的工具工作流被展开为多次模型可见的决策过程，消耗上下文并迫使模型在轨迹中管理低层次的数据流。我们提出了**HyperTool**，这是一种统一的可执行MCP风格工具接口，改变了模型可见的工具执行单元。模型通过一个代码块调用HyperTool，该代码块可以通过原始模式调用现有工具、操作返回值，并在本地传递中间结果，从而将确定性的工具子程序折叠为一次外部调用。为了训练模型使用此接口，我们从跨工具组合任务中合成了HyperTool格式的轨迹，并在真实的MCP环境中验证了它们。在MCP-Universe上，HyperTool使Qwen3-32B的平均准确率从15.69%提升至35.29%，Qwen3-8B从9.93%提升至33.33%，并在平均准确率上超越了GPT-OSS和Kimi-k2.5，表明我们的HyperTool可以显著提升多步骤工具使用的性能。 核心贡献：HyperTool通过引入一种新的统一工具接口，将多个确定性工具调用折叠为单个外部调用，显著提升了多步骤工具使用的效率与准确性。

EurekAgent: Agent Environment Engineering is All You Need For Autonomous Scientific Discovery
👥 Amy Xin, Jiening Siow, Junjie Wang
基于大语言模型（LLM）的智能体在自动化科学发现方面展现出越来越大的潜力。只要给定一个可优化的指标和执行环境，它们就能提出、验证并迭代科学解决方案，并且已经产生了优于人类设计方法的结果。随着模型能力的持续提升，我们认为自主科学发现的瓶颈正从规定智能体的工作流程转向设计智能体环境：即塑造智能体行为的资源、约束和接口。我们将这一问题框架化为“环境工程”：构建能够增强有益行为（如开放式的探索、系统化的成果管理以及智能体之间的协作）同时抑制有害行为（如奖励黑客攻击和高摩擦的人类监督）的环境。我们提出了 EurekAgent，这是一个面向指标驱动型自主科学发现的环境工程智能体系统。EurekAgent 从四个维度对环境进行了工程化处理：权限工程用于限制智能体的执行范围和隔离评估；成果工程用于文件系统和基于 Git 的协作；预算工程用于预算感知的探索；以及人机交互工程用于简化人类的监督与干预。EurekAgent 在多个数学、内核工程和机器学习任务上取得了新的最先进成果，包括以不到11美元的总API成本发现的26个圆的新最优打包结果。我们开源了代码和实验结果，并呼吁将环境工程作为开发可靠自主研究智能体的核心研究方向。 核心贡献：EurekAgent 提出了一种通过环境工程来增强自主科学发现智能体性能的方法，在多个领域实现了最先进的成果。

Operadic consistency: a label-free signal for compositional reasoning failures in LLMs
👥 Nathaniel Bottman, Yinhong Liu, Kyle Richardson
在没有真实标签的情况下检测大语言模型（LLM）推理失败，已经推动了多种置信度基准的发展，包括自我一致性、语义熵和P(True)，这些方法通常基于问题内部的采样与自评估。操作子理论（operad theory），即通过迭代替换构建系统的形式化理论，提出了一种互补的诊断方式：对于一个组合查询，模型直接给出的答案应与其通过对该查询声明分解后组合得出的答案一致。我们将这一想法具体化为“操作子一致性”（Operadic Consistency, OC），这是一种针对每个问题的信号。在四个多跳问答数据集上的十二个经过指令微调的LLM（参数范围从4B到671B，包含开源和闭源模型）中，OC与所有数据集的准确率均高度相关（皮尔逊相关系数 $r \in [0.86, 0.94]$，所有 $p \leq 0.0004$），并且是我们在所有四个数据集中唯一一个在所有情况下 $r \geq 0.85$ 的信号。链式思维自我一致性（CoT-SC；Wang et al., 2023）在HotpotQA和DROP上与OC相当（$r = 0.93, 0.87$），但在MuSiQue和StrategyQA上下降至 $r \approx 0.45$。在每个问题层面，OC提供了超越CoT-SC和语义熵的信息（聚类稳健性 $p \leq 10^{-16}$ 对于OC系数），且这一结论在控制构造的分解感知基线时依然稳健（$p \leq 10^{-13}$）。相同的信号在固定覆盖范围下的选择性预测性能方面优于调整后的CoT-SC基线，在等成本预算 $K = 3$ 下实现了显著提升（AUARC 提升 +0.086 至 +0.096，AUROC 提升 +0.092 至 +0.164；所有单元格的95%置信区间均不包含零）。在五个前沿思考模型上，当分解是从模型自身的链式思维中提取时，同样的等成本比较在所有16个（数据集，预算，指标）单元格中均表现出正的选择性预测点估计提升，其中16个中有12个的95%置信区间排除零。 核心贡献：本文提出了“操作子一致性”（Operadic Consistency, OC）作为一种无需标签即可检测LLM组合推理失败的新信号，并验证其在多个模型和数据集上的有效性，显著优于现有方法。

Recursive Agent Harnesses
👥 Elias Lumer, Sahil Sen, Kevin Paul
Recursive language models (RLMs) showed that recursion over model calls is an effective strategy for long-context reasoning, and production coding agents have begun to write code that spawns subagents at scale, most recently in Anthropic's dynamic workflows. We name and study the pattern between these two lines of work, where the recursive unit is a full agent harness with filesystem tools, code execution, and planning rather than a model call with no tools. We call this the Recursive Agent Harness (RAH) and frame it as harness recursion, the code-first extension to the model recursion of RLMs. A parent agent generates and runs an executable script that spawns subagent harnesses in parallel for fine-grained workloads and uses structured function calls for small subtasks. We provide a controlled evaluation on long-context reasoning. With the backbone held fixed at GPT-5 to match the published Codex and RLM baselines, RAH improves the Codex coding-agent baseline from 71.75% to 81.36% on Oolong-Synthetic (199 samples, 13 context-length buckets up to 4M tokens), a gain attributable to the harness rather than the model. With a stronger backbone, Claude Sonnet 4.5, the same design reaches 89.77%.
