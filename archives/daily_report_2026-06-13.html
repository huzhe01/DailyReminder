Operadic consistency: a label-free signal for compositional reasoning failures in LLMs
👥 Nathaniel Bottman, Yinhong Liu, Kyle Richardson
在没有真实标签的情况下检测大语言模型（LLM）推理失败，已经推动了多种置信度基准的发展，包括自我一致性、语义熵和P(True)，这些方法通常基于问题内部的采样与自评估。操作子理论（operad theory），即通过迭代替换构建系统的形式化理论，提出了一种互补的诊断方式：对于一个组合查询，模型直接给出的答案应与其通过对该查询声明分解后组合得出的答案一致。我们将这一想法具体化为“操作子一致性”（Operadic Consistency, OC），这是一种针对每个问题的信号。我们在四个多跳问答数据集上对十二个指令微调的大语言模型（参数范围从4B到671B，包含开源和闭源模型）进行了实验。结果显示，OC在所有数据集中均与准确率高度相关（皮尔逊相关系数 $r \in [0.86, 0.94]$，所有 $p \leq 0.0004$），并且是我们评估中唯一一个在所有四个数据集中均达到 $r \geq 0.85$ 的信号。链式思维自我一致性（CoT-SC；Wang et al., 2023）在HotpotQA和DROP上与OC表现相当（$r = 0.93, 0.87$），但在MuSiQue和StrategyQA上下降至$r \approx 0.45$。在每个问题层面，OC在所有数据集中提供了超出CoT-SC和语义熵的信息（聚类稳健性$p \leq 10^{-16}$用于OC系数），且这一结论在控制其他构造分解感知基线后依然稳健（$p \leq 10^{-13}$）。同样的信号在固定覆盖范围下相比调优后的CoT-SC基线提升了选择性预测性能（在相等成本预算$K = 3$下，AUARC提升+0.086至+0.096，AUROC提升+0.092至+0.164；95%置信区间排除零）。在五个前沿思考模型上，当分解从模型自身的链式思维中提取时，相同的相等成本比较在所有16个（数据集、预算、指标）单元格中均显示出正向的选择性预测点估计提升，其中16个中有12个的95%置信区间排除零。 核心贡献：本文提出了“操作子一致性”（Operadic Consistency, OC），一种无需真实标签即可有效检测大语言模型推理失败的新信号，并验证其在多个数据集和模型上的强相关性和鲁棒性。

Operads for compositional reasoning in LLMs
👥 Nathaniel Bottman, Kyle Richardson
摘要：问题分解，即将复杂查询拆分为更简单的子查询，并将子问题的答案组合起来生成最终答案，是一种广泛用于提升大语言模型（LLM）推理能力的策略，但目前缺乏严谨的数学基础。在本文中，我们提出使用操作元（operads），即能够建模多输入单输出操作及其组合的数学结构，作为描述问题分解的自然框架。我们定义了一个问题操作元 $Q$，其中的操作对应于问题模板，组合则对应于子答案的替换，并展示了问答模型如何可以被解释为 $Q$ 上的代数。除了重新表述现有实践外，这种操作元视角还指出了新的方法，特别是“操作元一致性”的概念，它衡量一个问答模型在其问题分解树的部分坍缩之间是否保持答案一致。我们在另一篇论文（Bottman, Liu, and Richardson, 2026）中对操作元一致性进行了实证评估，结果表明其与十二个LLMs和四个多跳问答数据集上的准确性高度相关，并优于基于温度的标准自一致性基线。我们认为，操作元是问题分解的自然数学基础，而诸如操作元一致性等不变量为分析和提升多步骤推理的可靠性开辟了新方向。 核心贡献：本文首次将操作元引入LLM的推理过程建模，提出了“操作元一致性”这一新指标，为分析和提升多步骤问答系统的可靠性提供了理论框架和实证支持。

EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments
👥 Jundong Xu, Qingchuan Li, Jiaying Wu
大型语言模型（LLM）代理在各种基准测试中表现出色，但大多数评估假设环境是静态的。相比之下，现实世界中的部署本质上是动态的，要求代理不断调整其知识、技能和行为以适应变化的环境和更新的任务条件。为了解决这一差距，我们引入了EvoArena，这是一个将环境变化建模为终端、软件和社会领域中逐步更新序列的基准套件。我们进一步提出了EvoMem，这是一种基于补丁的记忆范式，通过结构化的更新历史记录记忆的演变，使代理能够通过记忆的变化推理环境的演化。实验表明，当前的代理在EvoArena上表现不佳，在演化的终端、软件和社会偏好领域平均准确率为39.6%。EvoMem持续提升了性能，在EvoArena上平均提高了1.5%，同时也在标准基准GAIA和LoCoMo上分别提高了6.1%和4.8%。除了单个任务外，EvoMem还使EvoArena上的链级准确率提高了3.7%，其中成功需要连续完成一系列相关的进化子任务。机制分析表明，EvoMem改善了记忆中的证据捕获，表明更好地保留了完整的演化环境状态。我们的结果强调了在评估和记忆中建模演化对于可靠代理部署的重要性。 核心贡献：本文提出了EvoMem，一种基于补丁的记忆范式，用于追踪和利用记忆演化，从而提升大语言模型代理在动态环境中的鲁棒性与任务表现。

Influcoder: Distilling Decoders' Gradient Influence Rankings into an Encoder for Data Attribution
👥 Dimitri Kachler, Damien Sileo, Pascal Denis
随着大型语言模型（LLMs）能力的增长，人们越来越重视通过过滤训练数据中的样本来构建高质量的数据集。一般来说，数据归因（Data Attribution, DA）方法旨在估计训练数据集中单个样本如何预设模型生成特定输出的能力。例如，人们可能想知道哪些数据样本在训练后可能导致了模型的有毒行为。许多方法通过影响函数范式来量化这种条件关系。尽管这类方法在功能上是有效的，但它们缺乏足够的处理速度和存储紧凑性，难以在大规模数据集上实际应用。我们提出了一种名为 Influcoder 的方法，它是一种快速且经济高效的大规模基于影响的数据归因方法。 核心贡献：Influcoder 提出了一种高效的、可扩展的基于影响函数的数据归因方法，解决了传统方法在处理大规模数据时的速度和存储瓶颈问题。

HyperTool: Beyond Step-Wise Tool Calls for Tool-Augmented Agents
👥 Yaxin Du, Yifan Zhou, Yujie Ge
工具增强的LLM代理通常依赖于逐步的原子工具调用，其中每次调用、观察和值传递都会在主推理轨迹中被显式暴露。这导致了一个“执行粒度不匹配”问题：局部确定性的工具工作流被展开为多次模型可见的决策过程，消耗了上下文并迫使模型在轨迹中管理低级数据流。我们引入了**HyperTool**，一种统一的可执行MCP风格工具接口，该接口改变了模型可见的工具执行单位。模型通过一个代码块调用HyperTool，该代码块可以通过原始模式调用现有工具、处理返回值并在本地传递中间结果，从而将确定性的工具子程序折叠为单个外部调用。为了训练模型使用这一接口，我们从跨工具组合任务中合成HyperTool格式的轨迹，并在真实的MCP环境中验证它们。在MCP-Universe上，HyperTool使Qwen3-32B的平均准确率从15.69%提升至35.29%，Qwen3-8B从9.93%提升至33.33%，并且在平均准确率上超越了GPT-OSS和Kimi-k2.5，表明我们的HyperTool可以显著提升多步骤工具使用的性能。 核心贡献：HyperTool通过引入统一的可执行工具接口，显著提升了多步骤工具使用的效率和准确性。

EurekAgent: Agent Environment Engineering is All You Need For Autonomous Scientific Discovery
👥 Amy Xin, Jiening Siow, Junjie Wang
LLM-based agents have shown increasing potential in automating scientific discovery. Given an optimizable metric and an execution environment, they can propose, validate, and iterate scientific solutions, and have produced results that outperform human-designed approaches. As model capabilities continue to improve, we argue that the bottleneck for autonomous scientific discovery is shifting from prescribing agent workflows to designing agent environments: the resources, constraints, and interfaces that shape agent behavior. We frame this as environment engineering: building environments that amplify productive behaviors, such as open-ended exploration, systematic artifact management, and inter-agent collaboration, while suppressing harmful behaviors, such as reward hacking and high-friction human oversight. We present EurekAgent, an environment-engineered agent system for metric-driven autonomous scientific discovery. EurekAgent engineers the environment along four dimensions: permissions engineering for bounded agent execution and isolated evaluation; artifact engineering for filesystem and Git-based collaboration; budget engineering for budget-aware exploration; and human-in-the-loop engineering for easy human supervision and intervention. EurekAgent sets new state-of-the-art results on multiple mathematics, kernel engineering, and machine learning tasks, including new state-of-the-art 26-circle packing results discovered with less than $11 in total API cost. We open-source our code and results, and call for environment engineering as a core research direction for developing reliable autonomous research agents.
