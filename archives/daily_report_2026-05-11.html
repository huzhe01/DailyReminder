LLMs Improving LLMs: Agentic Discovery for Test-Time Scaling
👥 Tong Zheng, Haolin Liu, Chengsong Huang
测试时扩展（Test-Time Scaling, TTS）已成为通过在推理过程中分配额外计算资源来提高大语言模型性能的有效方法。然而，现有的TTS策略大多是手工设计的：研究人员依靠直觉手动设计推理模式并调整启发式规则，导致计算资源分配的空间大部分未被探索。我们提出了一种环境驱动的框架AutoTTS，该框架改变了研究者的设计对象：从单独的TTS启发式规则转向可以自动发现TTS策略的环境。AutoTTS的关键在于环境构建：发现环境必须使控制空间易于处理，并为TTS搜索提供低成本、频繁的反馈。作为具体实现，我们将宽度-深度TTS形式化为对预先收集的推理轨迹和探测信号进行控制器合成的问题，其中控制器决定何时分支、继续、探测、剪枝或停止，并且可以在无需重复调用LLM的情况下进行廉价评估。我们进一步引入了beta参数化方法以使搜索可处理，并采用细粒度执行跟踪反馈以通过帮助智能体诊断TTS程序失败原因来提高发现效率。在数学推理基准上的实验表明，所发现的策略在整体准确率与成本权衡方面优于强大的手工设计基线。这些策略能够推广到保留的基准任务和不同模型规模上，而整个发现过程仅需39.9美元和160分钟。我们的数据和代码将开源发布于https://github.com/zhengkid/AutoTTS。 核心贡献：本文提出了一个自动化发现TTS策略的框架AutoTTS，通过环境驱动的方法显著提升了大语言模型推理过程中的性能与成本平衡。

The Memory Curse: How Expanded Recall Erodes Cooperative Intent in LLM Agents
👥 Jiayuan Liu, Tianqin Li, Shiyi Du
摘要：上下文窗口的扩展通常被视为LLM能力的直接提升，但我们发现它在多智能体社会困境中系统性地失败。在7个LLM和4个游戏共500轮的实验中，28个模型-游戏设置中有18个在历史记录扩展后合作性下降，我们称这一现象为“记忆诅咒”。通过三项分析，我们揭示了其背后的机制。首先，对378,000条推理轨迹进行词汇分析表明，这种崩溃与前瞻意图的削弱有关，而非单纯的猜疑增加。我们通过目标微调验证了这一点：一个仅在前瞻轨迹上训练的LoRA适配器缓解了性能衰减，并能零样本迁移到不同游戏中。其次，在保持提示长度不变的情况下用合成的合作记录替换可见的历史记录（记忆净化），显著恢复了合作性，证明触发因素是记忆内容本身，而不仅仅是长度。最后，去除显式的Chain-of-Thought推理往往可以减少崩溃，表明反思反而会放大“记忆诅咒”。总体而言，这些结果重新定义了记忆作为多智能体行为的关键决定因素：更长的记忆回溯可能根据引发的推理模式破坏或支持合作。 核心贡献：该研究首次揭示了大语言模型中“记忆诅咒”现象，即记忆扩展并非总是有益，反而可能破坏多智能体间的合作，同时提出了通过控制记忆内容和推理方式来缓解这一问题的方法。

CA-SQL: Complexity-Aware Inference Time Reasoning for Text-to-SQL via Exploration and Compute Budget Allocation
👥 James Petullo, Nianwen Xue
尽管推理时学习的最新进展提高了大型语言模型（LLM）在文本到SQL任务中的推理能力，但当前的方法在Bird-Bench（BIRD）基准中最具挑战性的任务上仍表现不佳。这是由于对解决方案空间的探索不足，而这种探索是发现有潜力的候选查询并进一步优化以生成正确输出所必需的。为了解决这一问题，我们提出了CA-SQL，一种新颖的文本到SQL流水线方法，该方法利用任务难度估计动态调整探索范围以生成候选解决方案。此外，我们采用了一种基于进化搜索原理的定制提示种子方法，以进一步激发基础LLM的探索行为，并引入了一种新的投票方法来选择最终的最佳候选方案。实验表明，我们的方法仅使用GPT-4o-mini就在BIRD开发集“具有挑战性”的任务层面上取得了51.72%的最先进成绩，优于其他上下文学习方法，甚至包括那些使用更大模型的方法。总体而言，我们的方法在BIRD开发数据集上实现了具有竞争力的61.06%执行准确率和68.77%软F1分数。 核心贡献：提出了一种基于任务复杂度感知的文本到SQL推理方法CA-SQL，在不依赖大模型的情况下显著提升了在高难度任务上的性能表现。

Uncertainty-Aware Structured Data Extraction from Full CMR Reports via Distilled LLMs
👥 Yi Yu, Parker Martin, Zhenyu Bu
将自由文本的心脏磁共振（CMR）报告转换为可审计的结构化数据，仍然是队列构建、纵向管理和临床决策支持中的一个瓶颈。我们提出了 CMR-EXTR，这是一个轻量级框架，能够将自由文本的 CMR 报告转换为结构化数据，并为每个字段分配置信度以进行质量控制。通过教师-学生蒸馏流程，实现了完全离线推理并减少了人工标注的需求。不确定性评估整合了三种互补原则——分布合理性、采样稳定性和字段间一致性，用于确定是否需要人工复审。实验表明，CMR-EXTR 在变量级别实现了 99.65% 的准确率，展示了其可靠的提取能力和有信息量的置信度评分。据我们所知，这是首个集成了置信度估计的专用 CMR 提取系统。 核心贡献：提出了一种基于知识蒸馏和不确定性评估的轻量级 CMR 报告结构化提取系统 CMR-EXTR，首次实现了高精度且具备可信度评分的 CMR 数据自动化处理。

Fast Byte Latent Transformer
👥 Julie Kallini, Artidoro Pagnoni, Tomasz Limisiewicz
近期的字节级语言模型（LMs）在不依赖子词词汇表的情况下，其性能已可与基于词元的语言模型相媲美，但其实际应用受到缓慢的逐字节自回归生成过程的限制。我们通过引入新的训练和生成技术，在字节潜在变换器（BLT）中解决了这一瓶颈。首先，我们提出了 BLT 扩散（BLT-D），这是我们的最快 BLT 变体，并结合了标准的下一个字节预测损失与辅助的块级扩散目标进行训练。这使得推理过程中可以在每个解码步骤中并行生成多个字节，从而大幅减少生成序列所需的前向传播次数。其次，我们提出了两种受推测性解码启发的扩展方法，以牺牲部分速度为代价换取更高的生成质量：BLT 自我推测（BLT-S），其中 BLT 的局部解码器会继续在其正常块边界之后生成草案字节，随后通过一次完整的模型前向传播进行验证；以及 BLT 扩散+验证（BLT-DV），该方法在扩散生成后增加了一个自回归验证步骤来增强 BLT-D。所有方法在生成任务上的估计内存带宽成本都比原始 BLT 降低了超过 50%。每种方法都有其独特的优势，共同消除了字节级语言模型实用化的关键障碍。 核心贡献：本文提出了一种高效的字节级语言模型——Fast Byte Latent Transformer，通过扩散机制和推测性解码技术显著提升了生成速度并降低了计算成本。

GLiGuard: Schema-Conditioned Classification for LLM Safeguard
👥 Urchade Zaratiana, Mary Newhauser, George Hurn-Maloney
Ensuring safe, policy-compliant outputs from large language models requires real-time content moderation that can scale across multiple safety dimensions. However, state-of-the-art guardrail models rely on autoregressive decoders with 7B--27B parameters, reformulating what is fundamentally a classification problem as sequential text generation, a design choice that incurs high latency and scales poorly to multi-aspect evaluation. In this work, we introduce \textbf{GLiGuard}, a 0.3B-parameter schema-conditioned bidirectional encoder adapted from GLiNER2 for LLM content moderation. The key idea is to encode task definitions and label semantics directly into the input sequence as structured token schemas, enabling simultaneous evaluation of prompt safety, response safety, refusal detection, 14 fine-grained harm categories, and 11 jailbreak strategies in a single non-autoregressive forward pass. This schema-conditioned design lets supported task and label blocks be composed directly in the input schema at inference time. Across nine established safety benchmarks, GLiGuard achieves F1 scores competitive with 7B--27B decoder-based guards despite being 23--90$\times$ smaller, while delivering up to 16$\times$ higher throughput and 17$\times$ lower latency. These results suggest that compact bidirectional encoders can approach the accuracy of much larger guard models while drastically reducing inference cost. Code and models are available at https://github.com/fastino-ai/GLiGuard.
