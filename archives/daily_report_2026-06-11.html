Doc-to-Atom: Learning to Compile and Compose Memory Atoms
👥 Xingjian Diao, Wenbo Li, Yashas Malur Saidutta
长输入序列在大型语言模型的文档理解和多步推理中至关重要，但注意力机制的二次成本使得推理过程既占用大量内存又速度缓慢。上下文蒸馏通过将上下文信息压缩到模型参数中来缓解这一问题，而诸如 Doc-to-LoRA 的最新工作则将上下文蒸馏整合到一次前向传递中，为每个文档生成一个 LoRA 适配器。然而，为所有查询生成单一的适配器会导致无关查询干扰、有限的组合回忆能力以及难以扩展至长文档推理的问题。为了解决这些挑战，我们提出了 Doc-to-Atom（Doc2Atom），这是一种组合式参数化记忆框架，将每个文档分解为语义类型的知识原子。每个原子被编译为一个独立的微小 LoRA 适配器和一个溯源检索键。在推理时，一个轻量级的查询路由器仅选择并组合相关的原子以形成特定于查询的适配器，然后将其注入冻结的基础模型中。整个系统通过多目标蒸馏框架进行端到端训练。在六个多样化的问答基准上的实验表明，Doc2Atom 在性能上优于 Doc-to-LoRA 基线，同时降低了文档内部化的内存开销。 核心贡献：Doc2Atom 提出了一种基于知识原子的组合式参数化记忆框架，有效解决了长文档处理中的内存瓶颈和推理效率问题。

System Report for CCL25-Eval Task 5: New Dataset and LoRA-Fine-Tuned Qwen2.5
👥 Haotao Xie
近期，大语言模型（LLMs）在古典汉语翻译和古典诗歌生成领域取得了显著进展。然而，在古典诗歌的精准翻译和情感语义理解方面的研究仍较为有限。主要挑战在于大多数研究将诗歌赏析任务视为通用领域的任务，忽视了诗歌赏析的独特特征，而高质量、面向特定领域的数据集则极为稀缺。为了解决这一问题，我们将该任务分解为三个子任务：术语解释、语义解释和情感推理。基于多个开源数据集，我们进行了数据清洗与对齐，构建了一个专门用于该领域的《古诗文指令对数据集》（CCPoetry-49K），包含49,404个高质量的指令-响应对。随后，我们通过应用低秩适配（LoRA）技术对Qwen2.5-14B模型进行微调，提出了一个专注于该领域的专用大语言模型PoetryQwen。在CCL25-Eval Task 5基准上的实验结果表明，PoetryQwen得分为0.757，比基线模型Qwen2.5-14B-Instruct（0.690）提升了9.7%。这些结果明确表明，PoetryQwen在古典诗歌的精准翻译和情感理解方面显著提升了性能。我们提出的新数据集和方法论旨在支持大语言模型在特定领域的优化。 核心贡献：本文提出了针对古典诗歌领域优化的大语言模型PoetryQwen，并构建了高质量的CCPoetry-49K数据集，显著提升了诗歌翻译与情感理解的效果。

Which Models Are Our Models Built On? Auditing Invisible Dependencies in Modern LLMs
👥 Sanjay Adhikesaven, Haoxiang Sun, Sewon Min
现代大语言模型（LLM）的训练流程越来越多地依赖其他模型来生成数据、过滤语料库、评估输出以及指导开发决策。这些依赖关系是递归的：一个模型可能依赖于上游工件，而该工件本身的依赖关系仅在不同的发布版本和资源中有所记录。因此，完整的依赖结构被分散在各种异构的公共资源中，其复杂性和递归深度远远超出了人类追踪的能力。我们提出了 ModSleuth，这是一个智能系统，能够通过基于源代码的证据递归地重建 LLM 的依赖图谱。我们发现，当前的主要挑战不再是信息提取，而是如何定义“依赖关系”并协调不一致文档中的工件引用。我们通过一种形式化方法来应对这些挑战，该方法区分直接和间接依赖关系，通过以操作为中心的关系表示异构的流水线角色，并解决跨名称、版本和仓库的工件身份问题。我们将 ModSleuth 应用于四个包含大量公开资源的 LLM 发布版本，成功恢复了 1,060 个经过源码验证的依赖关系，并构建了现代 LLM 开发的大规模依赖图。这些图揭示了多跳许可义务、训练与评估耦合、发布时与训练时工件之间的差异，以及原本难以发现的文档不一致性。我们开源了 ModSleuth 和所生成的依赖图，以支持对现代 LLM 背后日益复杂的生态系统进行透明分析。 核心贡献：本文提出 ModSleuth 系统，首次实现了从异构公共资源中自动重构现代大语言模型的依赖图，揭示了模型开发中的隐藏依赖和潜在风险。

Verifiable Environments Are LEGO Bricks: Recursive Composition for Reasoning Generalization
👥 Hao Xiang, Qiaoyu Tang, Le Yu
可验证环境已成为增强大型语言模型（LLMs）推理能力的一种强大方法。尽管先前研究表明增加环境数量可以提升强化学习（RL）的性能，但现有的手动或单独构建方法受到线性扩展限制，从而阻碍了推理泛化的可扩展性。本文提出了RACES（**递归自动环境扩展框架**），该框架将可验证环境视为可组合的基本模块，并能够递归组装。其核心思想在于：当一个环境的输出类型与另一个环境的输入类型匹配时，它们可以自动融合为一个新的可验证环境，从而实现递归组合。RACES实现了300个独立环境，并定义了一组组合操作符（SEQUENTIAL、PARALLEL、SORT 和 SELECT），以诱导多样化的推理模式。大量实验表明，在这些复合环境中进行强化学习训练能够持续提升推理泛化能力。具体而言，RACES使DeepSeek-R1-Distill-Qwen-14B在六个未见过的基准上平均提升了3.1分（从48.2到51.3），并将Qwen3-14B的性能从58.8提升至61.1。此外，RACES仅使用50个基础环境即可达到与训练300个独立环境相当的性能，显示出环境利用效率的显著优势。 核心贡献：提出RACES框架，通过递归组合可验证环境来高效提升大型语言模型的推理泛化能力。

Breaking Entropy Bounds: Accelerating RL Training via MTP with Rejection Sampling
👥 Yucheng Li, Huiqiang Jiang, Yang Xu
强化学习（RL）已成为现代大型语言模型中的关键组成部分，但rollout阶段仍然是RL训练流水线中的主要瓶颈。尽管多标记预测（MTP）通过推测解码为加速rollouts提供了自然的解决方案，但许多研究发现，在RL训练过程中MTP的接受率显著下降，导致加速效果有限。为了解决这一瓶颈，我们提出了Bebop，对LLM后训练中的MTP进行了系统研究，并提供了将MTP集成到大规模RL流水线中的实用方法。首先，我们揭示了MTP接受率本质上受到模型熵波动的限制，这与RL阶段中熵值上升呈现明显的负线性关系。其次，我们表明，与贪婪草案采样相比，概率拒绝采样在很大程度上缓解了熵在RL中引入的干扰。我们进一步识别出传统MTP训练目标（交叉熵或KL散度）在这种情况下是次优的，因此我们提出了一种新颖的端到端TV损失函数，直接优化多步拒绝采样的接受率，从而实现了约10%的接受率提升，在数学推理、代码生成和智能体任务中达到最高95%的接受率，并额外获得最高25%的推理吞吐量提升。第三，我们在RL过程中测试了各种在线MTP训练策略，并表明使用端到端TV损失和拒绝采样的预RL MTP训练在整个RL过程中能够保持一致的接受率和加速效果，无需代价高昂的在线MTP更新。我们提供了广泛的实验和分析以验证我们的发现。实验结果表明，我们的方法在Qwen3.5、Qwen3.6和Qwen3.7模型的异步RL训练中实现了最高1.8倍的端到端加速。 核心贡献：本文系统地研究了MTP在RL训练中的应用，提出了一种基于端到端TV损失和拒绝采样的方法，有效提升了MTP的接受率和训练效率，实现了高达1.8倍的端到端加速。

ALIGNBEAM : Inference-Time Alignment Transfer via Cross-Vocabulary Logit Mixing
👥 Chirag Chawla, Pratinav Seth, Vinay Kumar Sankarapu
Domain fine-tuning degrades the safety of large language models: fine-tuned specialists readily comply with harmful prompts framed in domain language. Existing inference-time defenses that mix logits from a safe anchor model require both models to share a vocabulary, which rules them out for the cross-family specialists where safety is most degraded. We present ALIGNBEAM, a training-free method that lifts this restriction by translating anchor logits into the target model's vocabulary token-by-token at each decoding step; a small LLM judge then selects the safest among K candidate continuations. No weights are changed, and the safety-utility trade-off can be tuned at deployment without retraining. Across both cross-vocabulary and same-vocabulary evaluation pairs, ALIGNBEAM substantially raises refusal on adversarial benchmarks while keeping task accuracy and inference overhead within practical bounds. The results show that safety alignment can be transferred between model families at inference time, without touching either model's weights.

📊 广告与推荐算法

DeRes: Decoupling Residual Stability and Adaptivity for Scalable CTR Prediction
👥 Wenzhuo Cheng, Shipeng Nie, Qixin Guo
摘要：基于Transformer的点击率（CTR）预测模型在残差连接处面临日益严重的瓶颈：在预归一化（Pre-Norm）下，早期用户兴趣信号逐层被稀释；恒等跳接无法遗忘过时的兴趣；每一层只能看到其直接前驱层，丢失了长距离跨层依赖关系。最近基于注意力的残差变体（AttnRes）在语言模型中部分解决了这些问题，但去除了保护性的恒等跳接，并未在推荐系统中尝试。受双路径网络（DPN）和残差的HORNN视角启发，我们提出了DeRes，该方法通过两条并行路径处理每层——一条保持一阶特征重用和梯度流动的恒等残差路径，另一条通过关注所有先前块压缩输出以实现高阶回忆的块注意力残差路径。一个向量级门控机制为每个隐藏维度决定每条路径的权重。我们进一步提出点对注意力残差（Pointwise AttnRes），将跨层注意力中的Softmax替换为SiLU，使得多个过去的块可以同时被激活，无关块获得负权重（遗忘权重）——更符合CTR的并行多兴趣模式。在大规模工业数据集（3.31亿次交互来自主要社交媒体平台）、Criteo（4500万）和Avazu（4000万）上的实验表明，DeRes在计算开销增加不到5%的情况下，在AUC指标上优于包括OneTrans、TokenMixer-Large、UniMixer、mHC和AttnRes在内的十二个基线模型，最高提升达+0.32%。此外，DeRes拟合了一个明显更陡峭的计算-AUC扩展定律（gamma=0.118 vs. 0.071 for OneTrans，差距1.66倍），因此8层的DeRes性能可与16层的OneTrans相当，在相同AUC下节省约2倍的计算资源。消融实验证实了双路径设计优于任一单路径，恒等路径优于可学习残差路径，且SiLU优于Softmax。 核心贡献：本文提出了一种用于CTR预测的新型残差结构DeRes，通过分离残差稳定性和适应性，结合双路径设计和改进的注意力机制，在计算效率和模型性能之间取得了显著平衡。
