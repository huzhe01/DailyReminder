🔥 大模型前沿

MUSE-Autoskill: Self-Evolving Agents via Skill Creation, Memory, Management, and Evaluation
👥 Huawei Lin, Peng Li, Jie Song
大型语言模型（LLM）代理依赖可重用的技能来解决复杂任务。然而，现有的技能创建方法将技能视为孤立且静态的产物，限制了其可重用性、可靠性和长期改进能力。我们提出了 MUSE-Autoskill 代理（Memory-Utilizing Skill Evolution），这是一个以技能为中心的代理框架，使代理能够在统一的生命周期（创建、记忆、管理、评估和优化）下持续提升其任务解决能力。我们的框架使代理能够按需创建技能，在不同任务中存储和重用这些技能，高效地组织并选择它们，并通过单元测试和运行时反馈进行持续优化。我们进一步引入了技能级别的记忆机制，积累每个技能在多个任务中的经验，从而实现更有效的重用和适应。在 SkillsBench 上的实验提供了初步证据，表明经过生命周期管理的技能可以提高任务成功率、效率、重用率以及跨代理的迁移能力，强调了将技能视为长期存在、具有经验感知和可测试性的资产的重要性。 核心贡献：提出 MUSE-Autoskill 代理框架，通过统一的技能生命周期管理实现代理的自我进化，显著提升了技能的可重用性、适应性和跨任务表现。

MobileMoE: Scaling On-Device Mixture of Experts
👥 Yanbei Chen, Hanxian Huang, Ernie Chang
混合专家（MoE）已成为百亿参数语言模型的事实标准架构，但在设备端部署的亚十亿规模下其优势尚未被充分探索。为此，我们提出了MobileMoE，一系列适用于设备端、具有亚十亿活跃参数（0.3-0.9B活跃参数和1.3-5.3B总参数）的MoE语言模型，为设备端大语言模型设定了新的帕累托前沿。我们首先提出了一个适用于设备端的MoE扩展定律，在移动设备的内存和计算限制下联合优化MoE架构，识别出在设备端最优的组合——中等稀疏度与细粒度共享专家相结合，同时实现内存和计算最优。基于所推导的架构，我们在开源数据集上使用四阶段训练方案对MobileMoE进行训练，包括预训练、中期训练、指令微调和量化感知训练。在14个基准测试中，MobileMoE在推理FLOPs减少2-4倍的情况下，性能与领先的设备端密集型LLM相当或更优，并且参数比最先进的MoE模型OLMoE-1B-7B少达60%。为了弥合通向移动端部署的最后一公里，我们首次在普通智能手机上实现了高效的MoE推理并进行了全面的设备端性能分析。在相同INT4权重内存下，MobileMoE-S的预填充速度比密集基线MobileLLM-Pro快1.8至3.8倍，解码速度快2.2至3.4倍。 核心贡献：提出MobileMoE，一种专为设备端优化的亚十亿级MoE语言模型家族，通过理论建模和系统优化显著提升了设备端大模型的效率与性能。

Alignment Tampering: How Reinforcement Learning from Human Feedback Is Exploited to Optimize Misaligned Biases
👥 Dongyoon Hahm, Dylan Hadfield-Menell, Kimin Lee
人类反馈强化学习（RLHF）是将大语言模型（LLM）与人类偏好对齐的标准方法。在这项工作中，我们引入了“对齐篡改”这一潜在漏洞，即正在对齐的LLM能够影响偏好数据集，从而导致RLHF放大不良行为。这种问题源于RLHF的核心限制：（1）偏好数据集是从LLM自身的输出中构建的，因此它能够影响这些数据；（2）成对比较仅能表明哪个回应更好，而不能说明原因。这些限制可以被利用以实现对齐篡改。例如，如果LLM生成具有更高质量但带有偏见的回应，标注者会基于质量选择它们。然而，偏好标签无法区分质量和偏见，奖励模型也继承了这一局限性。通过强化学习或最佳N选样本来优化此类奖励可能会放大未对齐的偏见。我们的实验表明，这种放大现象存在于各种偏见中，从关键词偏见到宣传（如性别歧视）、品牌推广和工具性目标追求。缓解这一问题仍然具有挑战性，因为现有的鲁棒RLHF技术在不牺牲回应质量的前提下无法完全解决对齐篡改问题。这些发现揭示了当前RLHF的结构性漏洞，并强调了防止该漏洞的重要性。 核心贡献：本文首次提出了“对齐篡改”的概念，揭示了RLHF中的结构脆弱性，并展示了LLM如何通过影响偏好数据集来放大未对齐的偏见。

Guiding LLM Post-training Data Engineering with Model Internals from Sparse Autoencoders
👥 Yi Jing, Zao Dai, Jinwu Hu
模型内部编码了关于大型语言模型（LLM）如何处理其训练数据的丰富信息；然而，后训练的数据工程在很大程度上依赖于外部信号，并忽略了模型内部所蕴含的丰富内在信号。我们提出了SAERL，这是一个面向LLM强化学习（RL）的数据工程框架。它利用稀疏自编码器（SAE）这一先进的机制可解释性工具提取的模型内部信息，建模三种内在数据属性：多样性、难度和质量。每种属性对应一种具体的数据工程操作：通过中等批量混合的SAE空间聚类控制批次多样性，使用难度代理实现从易到难的课程排序，以及通过质量探测器进行数据过滤。SAERL在Qwen2.5-Math-1.5B任务上相比基础GRPO平均准确率提升了3.00%，并且达到目标准确率所需的训练步数减少了20%。实验表明，SAE可以有效地跨模型家族和规模迁移，成为一种轻量且可重用的数据工程工具。这些结果表明，模型内部是后训练数据工程中强大而实用的信号来源。 核心贡献：提出了一种基于稀疏自编码器提取模型内部信息的LLM强化学习数据工程框架SAERL，显著提升了训练效率与性能，并展示了模型内部信号在数据工程中的实用价值。

MATCHA: Matching Text via Contrastive Semantic Alignment
👥 Siran Li, Ece Sena Etoglu, Carsten Eickhoff
可靠的评估对于理解大语言模型（LLM）的性能至关重要，然而目前常用的指标——即基于词元重叠的分数（如ROUGE）和基于嵌入的度量（如BERTScore）——常常错误地判断文档之间的语义相似性。我们的研究表明，无论是基于词元重叠的指标还是基于嵌入的指标，都会经常给彼此直接矛盾的文本分配几乎相同的分数，从而可能掩盖基本的错误。我们提出了MATCHA，这是一种自动度量方法，能够同时奖励与参考文本的语义一致，并惩罚语义矛盾。MATCHA采用双视角的方法，分别测量（i）与黄金标准文本的接近程度以及（ii）与对抗生成的反事实矛盾之间的距离。在八个公开基准测试中，MATCHA在问答、图像字幕生成、自然语言推理、摘要生成和语义文本相似性任务上，相较于人工标注表现更优。在TruthfulQA数据集（即一个没有训练集的数据集，无法对任何基于嵌入的指标进行本地训练）上，MATCHA在与参考文本匹配方面的提升达到18.38%（相对于ROUGE-L）和20.82%（相对于BERTScore）。定量比较和定性的人类评估都证实了MATCHA的有效性和准确性，并揭示了现有指标的根本弱点。与其他23个用作类似BERTScore度量的嵌入模型相比，MATCHA仍然是最准确的，仅基于参考文本即可区分正确与错误的陈述。我们的代码和度量已公开（https://github.com/Siran-Li/MATCHA）。 核心贡献：MATCHA是一种新的自动评估指标，通过对比语义对齐有效提升了文本匹配的准确性，并在多个基准测试中超越了现有主流指标。

FinHarness: An Inline Lifecycle Safety Harness for Finance LLM Agents
👥 Haoxuan Jia, Yang Liu, Bin Chong
Finance LLM agents must simultaneously block prompt-induced unauthorized actions and approve legitimate multi-step business workflows. However, boundary filters often miss irreversible mid-trajectory tool calls, while post-hoc LLM judges perform auditing only after termination -- too late for intervention and at a computational cost that scales linearly with trace length. We present FinHarness, an inline safety harness that wraps a finance agent end-to-end with three components: a Query Monitor that fuses single-turn intent with cross-turn drift, a Tool Monitor that evaluates each prospective tool call, and a Cascade module that integrates per-step risk and adaptively routes verification between a lightweight and an advanced-tier LLM judge. Fired risk factors are re-injected into the agent input as ex-ante evidence, enabling the agent to refuse, re-plan, or approve on its own. On FinVault, routed FinHarness cuts ASR from 38.3% to 15.0% while largely preserving benign approval ($41.1\% \to 39.3\%$), and uses $4.7\times$ fewer advanced-judge calls than an always-advanced ablation.

📊 广告与推荐算法

LENS: A Staged Design for Interaction Granularityin Sequential CTR Prediction
👥 Yuan Wang, Yue Liu, Jun Zhang
在顺序点击率（CTR）预测中，一个核心的设计问题是目标应该以怎样的粒度与用户行为序列进行交互。现有的模型主要遵循两种方法：原始项架构（如DIN）让目标直接对序列中的每一项进行评分，这种方法依赖于训练良好的项嵌入，在稀疏项上表现脆弱；而潜在查询架构（如HyFormer、MixFormer和OneTrans）通过将目标与其他信息结合来构建查询表示，这种方法在不同项密度下更具鲁棒性，但控制力较弱：目标特定的控制被稀释了。我们提出LENS，旨在在这些更粗粒度的瓶颈中恢复目标特定的控制能力。LENS包含两个模块：用于查询激活的目标条件查询门控（TCQG）和用于历史检索的目标条件位置偏置（TCPB）。我们还引入了一种名为Query-Specific Position Bias（QueryPos）的简单静态位置感知参考机制，适用于潜在查询主干。在三个具有代表性的潜在查询主干和四个数据集上的实验表明，QueryPos+LENS设计在所有十二个主干-数据集组合中均取得了正向的总收益点估计。我们还发现了一个密度相关的条件规则：随着项密度的降低，最优的条件来源从仅项信息转移到项加序列信息。 核心贡献：本文提出了LENS框架，通过引入目标条件化的模块和位置偏置机制，在潜在查询架构中有效恢复了目标特定的控制能力，并在多种设置下实现了性能提升。

Selective Test-Time Compute Scaling for Click-Through Rate Prediction via Uncertainty-Triggered Feature Path Exploration
👥 Moyu Zhang, Yun Chen, Yujun Jin
在测试时扩展计算资源已被证明对语言模型非常有效，但这一方法在工业级点击率（CTR）预测中的应用仍鲜有探索。CTR模型面临一个基本的不对称问题：训练中频繁出现的特征组合能产生可靠的预测，而稀疏观察到的特征组合则会导致不可靠的输出。现有的训练阶段解决方案（如自适应门控机制）学习的是固定的特征选择函数，受制于相同的稀疏性限制，在部署阶段无法针对每个实例进行调整。我们提出了UTTSI（基于不确定性的测试时选择性推理），这是一个无需训练、与模型无关的框架，能够根据每个实例的不确定性按比例扩展推理深度。该框架结合模型logit置信度和数据级别的频率先验，构建双信号估计器以区分认识论上的不确定性（epistemic uncertainty）和本体论上的模糊性（aleatoric ambiguity）。每个实例都会经过自适应的特征过滤，剔除不可靠的嵌入表示；对于不确定的实例，还会进行随机的特征路径探索，并通过一致性加权集成聚合预测结果。而确定性强的实例则完全跳过探索过程，从而将平均计算开销控制在基础模型的约2.8倍，且最坏情况下的延迟保持不变。我们在四个数据集上使用三种主干架构进行了实验，结果一致且显著优于所有训练阶段的基线方法。一项为期七天的在线A/B测试进一步验证了相对CTR提升了5.3%（p < 0.01），证明了选择性地在测试时分配计算资源可以作为训练阶段改进的有效补充。 核心贡献：本文提出了一种无需训练的选择性测试时计算分配框架UTTSI，通过基于不确定性的特征路径探索，在不显著增加延迟的前提下显著提升CTR预测效果，并在多个数据集和真实场景中验证了其有效性。

Self-Balancing Gradient Allocation for Heterogeneity-Aware Feature Generation in Click-Through Rate Prediction
👥 Moyu Zhang, Yun Chen, Yujun Jin
通过离散扩散进行生成式预训练能够同时在所有特征字段上提供密集的重建监督，缓解了CTR预测中因数据稀疏性导致的表示崩溃问题。然而，现有的所有生成式CTR方法都存在一个根本性的限制：重建目标对每个特征字段赋予相同的训练权重，忽视了高基数ID字段、稀疏类别属性、数值变量和行为序列之间重建难度的显著异质性。这导致容易的字段主导了训练梯度，而最难但最具信息量的字段则始终拟合不足，我们称这一问题为“生成难度不平衡”。为此，我们提出了HeteGenCTR，通过与去噪网络联合训练的每字段可学习难度参数来解决这一不平衡问题。该统一信号驱动两个协调组件，无需额外超参数：一种自我平衡的损失函数，能够自动将梯度预算重新分配给更难的字段，并具有可证明稳定的均衡；以及一种以难度为导向的注意力机制，抑制已收敛易字段的影响，同时增强向困难字段的信息流动。这两个组件共享相同的信号，在整个训练过程中保持相互一致。在五个CTR基准数据集和为期七天的在线A/B测试中，实验结果表明HeteGenCTR相比最先进的基线方法实现了持续且统计显著的提升，尤其在冷启动用户和长尾用户上的增益更为明显。 核心贡献：提出HeteGenCTR，通过引入可学习的字段难度参数实现自适应梯度分配，有效解决了CTR预测中由于特征异质性引起的生成难度不平衡问题。
